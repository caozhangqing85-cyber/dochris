# 编译文档

编译是 dochris 的核心功能，使用 AI 将原始文件转化为结构化知识。

## 编译流程

```
Phase 2: Compilation
┌──────────────────────────────────────┐
│  1. 提取文本（PDF/音频/图片）         │
│  2. 分块（semantic chunking）         │
│  3. LLM 编译每个分块                  │
│     - 提取关键概念                    │
│     - 生成结构化摘要                  │
│     - 建立知识关联                    │
│  4. 层次化合并（Hierarchical Merge）  │
│  5. 质量评分（阈值 85）               │
│  6. 保存编译结果                      │
└──────────────────────────────────────┘
```

## 质量评估

编译完成后自动评分（0-100），评估维度：

- **完整性**：是否覆盖原文主要内容
- **准确性**：概念提取是否准确
- **结构化**：摘要是否清晰有条理
- **关联性**：是否建立了有效的知识关联

低于阈值（默认 85）的文档会被标记为需要重新编译。

## 并发编译

```bash
# 默认并发数（配置中设置）
kb compile

# 自定义并发数
kb compile --concurrency 4

# 限制编译数量
kb compile 10
```

## 编译输出

编译结果保存在 `outputs/` 目录：

```
outputs/
└── SRC-0001/
    ├── compiled.md      ← 主编译结果
    ├── concepts.json    ← 提取的概念列表
    └── metadata.json    ← 编译元数据
```
