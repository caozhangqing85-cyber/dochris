# 核心概念

了解 dochris 的设计理念和核心概念。

## 四阶段流水线

dochris 采用四阶段流水线处理知识：

```
源文件 → Phase 1 摄入 → Phase 2 编译 → Phase 3 查询 → Phase 4 分发
```

### Phase 1：摄入

扫描源文件目录，创建符号链接和 manifest：

- **文件发现**：递归扫描源目录中的 PDF、Markdown、音频、视频等文件
- **去重**：通过 SHA-256 哈希值识别重复文件
- **Manifest 生成**：为每个文件创建 JSON 元数据文件（`manifests/sources/SRC-NNNN.json`）
- **符号链接**：在 `raw/` 目录中创建指向源文件的链接

### Phase 2：编译

使用 AI 将原始文件转化为结构化知识：

1. **文本提取**：通过 markitdown 将 PDF、Word、代码等转为纯文本
2. **智能分块**：按语义边界将长文本切分为合适的片段
3. **LLM 编译**：对每个分块调用 LLM，提取关键概念、生成摘要
4. **层次化合并**：将多个分块的编译结果合并为完整文档
5. **质量评分**：自动评估编译结果质量（0-100 分）

### Phase 3：查询

通过自然语言搜索和查询知识库：

- **向量搜索**：使用嵌入模型进行语义相似度搜索
- **关键词搜索**：在概念和摘要中匹配关键词
- **LLM 增强**：查询结果经 LLM 二次处理，生成精准回答

### Phase 4：分发

将高质量知识分发到不同目标：

- **晋升到 Wiki**：质量达标的文档晋升到 `wiki/` 目录
- **Obsidian 同步**：双向同步 Obsidian Vault
- **知识图谱**：构建概念之间的关联图谱

## 四层信任模型

dochris 使用四层信任模型管理知识质量：

```
Layer 0 (outputs/)     → LLM 输出，未经验证
Layer 1 (wiki/)        → 质量达标（≥ 85 分），半可信
Layer 2 (curated/)     → 人工审核，可信
Layer 3 (locked/)      → 锁定，不可变
```

| 层级 | 目录 | 信任度 | 说明 |
|------|------|--------|------|
| L0 | `outputs/` | 未验证 | LLM 原始输出，默认存储位置 |
| L1 | `wiki/` | 半可信 | 通过质量评估（≥ 85 分） |
| L2 | `curated/` | 可信 | 经过人工审核确认 |
| L3 | `locked/` | 最高 | 锁定状态，不可修改 |

## 数据流

```
源文件 (HDD) → raw/ (symlinks) → manifests/ → outputs/ (L0)
                                                        ↓ (质量≥85)
                                                  wiki/ (L1) → Obsidian
                                                        ↓ (人工审核)
                                                  curated/ (L2)
                                                        ↓ (锁定)
                                                  locked/ (L3)
```

## 质量评分体系

编译结果自动评分（满分 100，晋升门槛 85 分）：

| 维度 | 分值 | 说明 |
|------|------|------|
| 摘要长度 | 0-35 | 800-1500 字符最佳 |
| 关键点完整性 | 0-40 | 4-5 个独立要点 |
| 学习价值 | 0-25 | 关键词密度（方法、策略、原则） |
| 信息密度 | 0-10 | 具体技术/工具密度 |
| 一句话质量 | 0-10 | 10-50 字符的精炼总结 |
| 概念完整性 | 0-20 | 3-5 个完整概念 |
| 模板检测 | -20 | 检测到模板文本扣分 |

## 插件系统

6 个扩展点支持自定义编译流程：

| Hook | 触发时机 | 用途 |
|------|----------|------|
| `ingest_parser` | 文件摄入时 | 自定义文件解析器 |
| `pre_compile` | 编译前 | 预处理、过滤 |
| `post_compile` | 编译后 | 后处理、通知 |
| `quality_score` | 质量评分时 | 自定义评分维度 |
| `pre_query` | 查询前 | 查询预处理 |
| `post_query` | 查询后 | 结果后处理 |

详见 [插件系统](../advanced/plugins.md)。
