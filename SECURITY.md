# 安全策略

## 支持的安全版本

当前支持的安全版本：
- 1.0.0 及以上

## 报告安全漏洞

如果你发现安全漏洞，**请不要**公开创建 Issue。

### 报告流程

1. **发送邮件**到：gongzhuming@example.com
   - 邮件标题：`[SECURITY] Dochris 漏洞报告`
   - 包含以下信息：
     - 漏洞描述
     - 影响版本
     - 复现步骤
     - 潜在影响
     - 建议的修复方案（如有）

2. **确认收到**
   - 我们会在 48 小时内确认收到你的报告
   - 提供预计的修复时间表

3. **修复过程**
   - 我们会调查并验证漏洞
   - 开发修复补丁
   - 在发布新版本前协调公开披露

### 响应时间表

| 严重级别 | 响应时间 | 修复时间 |
|---------|---------|---------|
| Critical | 48 小时 | 7 天 |
| High | 72 小时 | 14 天 |
| Medium | 1 周 | 30 天 |
| Low | 2 周 | 下个版本 |

### 严重级别定义

- **Critical**: 远程代码执行、无需认证的数据访问
- **High**: 需要认证的敏感数据访问、权限提升
- **Medium**: 本地信息泄露、有限的影响范围
- **Low**: 信息泄露影响小、需要复杂条件触发

## 安全最佳实践

### API 密钥管理

- **永远不要**将 API 密钥提交到代码仓库
- 使用环境变量或 `.env` 文件存储密钥
- 确保 `.env` 文件在 `.gitignore` 中
- 定期轮换 API 密钥

### 文件权限

- 确保工作区目录（`~/.knowledge-base`）权限正确
- 不要在公开仓库中上传包含敏感信息的 manifest

### LLM 内容过滤

系统会自动过滤敏感词，但：
- 不要处理包含个人身份信息的文档
- 不要上传包含机密信息的文件
- 注意 API 提供商的内容审核政策

## 已知安全注意事项

### 外部依赖

项目依赖以下外部服务，其安全策略由各自提供商负责：

- **OpenAI 兼容 API**: 文档内容会发送到 LLM 服务商
- **ChromaDB**: 向量存储在本地
- **HuggingFace 模型**: 本地运行，不发送数据

### 数据隐私

- 所有 LLM 处理通过 API 进行，受提供商隐私政策约束
- 向量嵌入存储在本地 ChromaDB 中
- 源文件路径以符号链接形式存储，不修改原始文件
